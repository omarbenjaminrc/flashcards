¿Cuál es la función principal del comando <b><code>docker run</code></b>?;Es el comando principal para <b>crear y iniciar un nuevo contenedor</b> a partir de una imagen. Si la imagen no está localmente, la descarga (pull) automáticamente.
En el comando <code>docker run docker/whalesay cowsay "Hola"</code>, ¿qué representan <code>docker/whalesay</code> y <code>cowsay "Hola"</code>?;<code>docker/whalesay</code> es la <b>imagen</b> a usar. <br><code>cowsay "Hola"</code> es el <b>comando específico</b> que se ejecutará <i>dentro</i> del contenedor una vez iniciado.
¿Qué tipo de tarea ejemplifica el contenedor <code>whalesay</code> y qué sucede cuando termina?;Ejemplifica una <b>tarea simple y efímera</b>. Una vez que el comando interno (<code>cowsay</code>) finaliza, el contenedor <b>se detiene automáticamente</b>.
¿Para qué se usa la opción <b><code>-e</code></b> en <code>docker run</code> (ej. <code>-e VAR=valor</code>)?;Se usa para pasar <b>Variables de Entorno (Environment Variables)</b> al contenedor, permitiendo configurarlo sin modificar la imagen (ej., <code>-e POSTGRES_PASSWORD=...</code>).
¿Para qué se usa la opción <b><code>-p</code></b> en <code>docker run</code> y cuál es su formato (ej. <code>-p 8080:80</code>)?;Se usa para <b>Mapear Puertos</b>, conectando un puerto de la máquina anfitriona (host) a un puerto dentro del contenedor. Formato: <code><b>puerto_anfitrión:puerto_contenedor</b></code>. Permite acceso externo al servicio del contenedor.
¿Qué significa especificar una imagen con un <b>tag</b> como en <code>postgres:15.1-alpine</code>?;El tag (<code>15.1-alpine</code>) permite seleccionar una <b>versión específica</b> o variante de la imagen base (<code>postgres</code>).
¿Qué tipo de aplicación ejemplifica el contenedor <code>postgres</code> ejecutado con <code>docker run</code>?;Ejemplifica un <b>servicio de larga duración</b> (como una base de datos) que generalmente permanece corriendo en segundo plano después de iniciarse.
¿Cómo se interactúa típicamente con un servicio corriendo dentro de un contenedor (como el de <code>postgres</code>) después de usar <code>docker run -p ...</code>?;Se utiliza un <b>cliente externo</b> (ej. pgAdmin, psql, un navegador web) que se conecta a través del puerto mapeado en la máquina <b>anfitriona</b> (ej. <code>localhost:5432</code>).
