
<p>¿Cuál es el comando principal para **crear e iniciar un nuevo contenedor** a partir de una imagen?</p>;<h3><code>docker run [OPCIONES] IMAGEN</code></h3><p>Crea un nuevo contenedor a partir de una imagen y lo inicia automáticamente.</p><p><strong>Opciones clave:</strong></p><ul><li><code>-d</code>: Detached (ejecutar en segundo plano).</li><li><code>-p HOST:CONTAINER</code>: Publicar/mapear puertos.</li><li><code>-v HOST/VOLUME:CONTAINER</code>: Montar volúmenes.</li><li><code>--name NOMBRE</code>: Asignar nombre al contenedor.</li><li><code>-it</code>: Interactivo + TTY (para shell).</li></ul><p><em>Analogy: Crear y empezar una casa desde un plano.</em></p>
<p>Quiero ver una lista de los contenedores que **están corriendo ahora mismo**. ¿Qué comando uso?</p>;<h3><code>docker ps</code></h3><p>Lista los contenedores que se encuentran en estado "Up" (en ejecución).</p><p><em>Analogy: Ver qué casas tienen las luces encendidas.</em></p>
<p>Necesito ver **TODOS los contenedores** que he creado, incluyendo los que están detenidos. ¿Qué comando es?</p>;<h3><code>docker ps -a</code></h3><p>Lista todos los contenedores, independientemente de su estado (Up, Exited, etc.).</p><p><em>Analogy: Ver la lista completa de casas construidas, activas o no.</em></p>
<p>¿Cómo detengo un contenedor que está en ejecución?</p>;<h3><code>docker stop CONTENEDOR</code></h3><p>Envía una señal de detención al contenedor especificado por su ID o nombre.</p><p><em>Analogy: Pedir a la gente que salga de la casa y apagar las luces.</em></p>
<p>Tengo un contenedor detenido (`Exited`) y quiero **iniciarlo de nuevo**. ¿Qué comando utilizo?</p>;<h3><code>docker start CONTENEDOR</code></h3><p>Inicia un contenedor que previamente fue detenido.</p><p><em>Analogy: Pedir a la gente que vuelva a la casa y encender las luces.</em></p>
<p>Quiero detener un contenedor y que automáticamente **vuelva a iniciarse**. ¿Hay un solo comando para eso?</p>;<h3><code>docker restart CONTENEDOR</code></h3><p>Detiene un contenedor en ejecución y luego lo inicia de nuevo. Útil para reinicios rápidos.</p><p><em>Analogy: Apagar y encender rápidamente una casa.</em></p>
<p>He terminado de usar un contenedor (está detenido) y quiero **eliminarlo por completo**. ¿Qué comando es?</p>;<h3><code>docker rm CONTENEDOR</code></h3><p>Elimina un contenedor específico.</p><p><strong>Importante:</strong> El contenedor **debe estar detenido** para poder eliminarlo. Usa <code>docker ps -a</code> para ver los detenidos.</p><p><em>Analogy: Demoler una casa que ya no usas.</em></p>
<p>Mi contenedor está corriendo pero creo que hay un error. Necesito ver la **salida estándar (logs)** del proceso principal del contenedor. ¿Cómo lo hago?</p>;<h3><code>docker logs CONTENEDOR</code></h3><p>Muestra los logs (stdout y stderr) generados por el contenedor. Puedes usar <code>-f</code> para seguirlos en tiempo real.</p><p><em>Analogy: Leer el registro de eventos dentro de la casa.</em></p>
<p>El contenedor ya está corriendo, pero necesito ejecutar un comando **dentro de él** (por ejemplo, abrir una shell o ejecutar un script de depuración). ¿Cuál es el comando?</p>;<h3><code>docker exec [OPCIONES] CONTENEDOR COMANDO</code></h3><p>Ejecuta un comando específico dentro de un contenedor que ya está en ejecución.</p><p><strong>Uso común:</strong> <code>docker exec -it CONTENEDOR /bin/bash</code> (para abrir una shell interactiva).</p><p><em>Analogy: Entrar en una casa habitada y pedirle a alguien que haga algo.</em></p>
<p>Quiero obtener una imagen de Docker Hub (o un registro) para usarla localmente. ¿Qué comando la descarga?</p>;<h3><code>docker pull IMAGEN[:TAG]</code></h3><p>Descarga una imagen desde un registro (por defecto Docker Hub) a tu máquina local.</p><p>El <code>:TAG</code> especifica la versión (ej: <code>ubuntu:20.04</code>). Si no se especifica, descarga <code>:latest</code>.</p><p><em>Analogy: Obtener un plano de la biblioteca de arquitectos.</em></p>
<p>Quiero ver una lista de **todas las imágenes** que tengo descargadas localmente en mi máquina. ¿Qué comando utilizo?</p>;<h3><code>docker images</code></h3><p>Lista las imágenes que están almacenadas localmente.</p><p>(También puedes usar <code>docker image ls</code>).</p><p><em>Analogy: Ver todos los planos que tienes guardados.</em></p>
<p>Ya no necesito una imagen descargada localmente. Quiero **eliminarla**. ¿Cuál es el comando?</p>;<h3><code>docker rmi IMAGEN</code></h3><p>Elimina una imagen local por su ID o nombre.</p><p><strong>Importante:</strong> Solo se puede eliminar una imagen si **ningún contenedor** (ni siquiera detenidos) depende de ella.</p><p><em>Analogy: Tirar a la basura un plano que ya no necesitas.</em></p>
<p>No sé si una imagen existe en Docker Hub o cómo se llama. Quiero **buscar imágenes** en el registro público. ¿Qué comando me ayuda?</p>;<h3><code>docker search TERMINO</code></h3><p>Busca imágenes en Docker Hub (o el registro configurado) que coincidan con el término dado.</p><p><em>Analogy: Buscar un plano en el catálogo de la biblioteca.</em></p>
